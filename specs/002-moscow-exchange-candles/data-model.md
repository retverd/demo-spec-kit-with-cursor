# Data Model: CandleRecord (LQDT/TQTF)

## Цель
Определить модель данных дневных свечей для инструмента **LQDT** на доске **TQTF**, используемую в сервисе `MoexClient`, валидаторе `validate_candles` и писателе `XLSXWriter`.

## Сущность: CandleRecord
- `date: date` — календарная дата свечи (`YYYY-MM-DD`), одна запись на день.
- `open: float | None` — цена открытия; допускается `None` при отсутствии данных.
- `high: float | None` — максимальная цена; допускается `None`.
- `low: float | None` — минимальная цена; допускается `None`.
- `close: float | None` — цена закрытия; допускается `None`.
- `volume: float | None` — объём торгов; допускается `None`.
- `instrument: str` — идентификатор инструмента, фиксированно `LQDT`.
- `board: str` — код доски, фиксированно `TQTF`.

## Инварианты и правила валидации
1. **Полнота периода**: список свечей должен покрывать все даты периода `[start_date, end_date]` (7 календарных дней).
2. **Уникальность дат**: не допускаются дубликаты дат.
3. **Диапазон дат**: каждая дата должна попадать в период запроса.
4. **Значения полей**:
   - `open`, `high`, `low`, `close`, `volume` — либо `None`, либо число `>= 0`.
   - Если значение присутствует, оно должно приводиться к `float`.
5. **Инструмент и доска**: `instrument` = `LQDT`, `board` = `TQTF`; отклонения считаются ошибкой данных.

## Маппинг из ISS‑API (см. research.md)
- `begin` → `date` (используется только дата)
- `open` → `open`
- `high` → `high`
- `low` → `low`
- `close` → `close`
- `volume` → `volume`
- константы: `instrument="LQDT"`, `board="TQTF"`
